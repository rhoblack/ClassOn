## 3. 실시간통신 에이전트 회의 결과

### Socket.io 룸 설계
- `course:{courseId}` — 해당 과정 학생 전체
- `teacher` — 강사 전용 룸

### 이벤트 프로토콜 확정

**학생 → 서버**
```
student:join          { enrollmentId, courseId }
student:feedback      { enrollmentId, courseId, buttonIndex, color, emoji, label }
student:question      { enrollmentId, courseId, text }  // 익명 처리
student:help-request  { enrollmentId, courseId }
student:quiz-answer   { sessionId, enrollmentId, answer }
```

**서버 → 강사**
```
teacher:student-joined      { enrollmentId, name, courseId }
teacher:student-left        { enrollmentId, courseId }
teacher:feedback-update     { enrollmentId, color, emoji, label, timestamp }
teacher:question-received   { id, text, timestamp }
teacher:help-requested      { enrollmentId, name, timestamp }
teacher:quiz-answer-update  { sessionId, answered, total, distribution }
```

**서버 → 학생 (broadcast)**
```
student:quiz-start    { sessionId, quiz: { type, question, options, timeLimit } }
student:quiz-end      { sessionId }
student:notice        { text, links: [{label, url}] }
student:mode-change   { mode: 'explain'|'practice'|'quiz' }
```

### 구현 원칙
- 서버가 피드백 이벤트 수신 시 DB 저장 + 강사에게 즉시 전달 (2-step)
- 학생 연결 해제 시 `teacher:student-left` 전송 (책상 색상 초기화)
- 퀴즈 응답은 Socket으로 수신 → DB 저장 → 집계 후 강사에게 전달
