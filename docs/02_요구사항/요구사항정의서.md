# ClassOn 요구사항 정의서

## 핵심 요구사항

### 1. 실시간 피드백
- 학생이 수업 중 버튼 클릭으로 이해도를 표현할 수 있어야 함
- 피드백은 **익명**으로 처리되어 학생이 솔직하게 반응할 수 있어야 함
- 강사는 학생들의 반응을 **실시간**으로 확인할 수 있어야 함
- 강사 화면에 교실 **책상 배치도**가 표시되고 책상 색상으로 상태를 한눈에 파악

### 2. 학생 개별 케어
- 강사가 학생 한 명 한 명의 특성과 실력을 파악할 수 있어야 함
- 학생 프로필, 누적 이해도, 퀴즈 성취도, 강사 메모를 통합 관리
- 책상 클릭 시 해당 학생의 모든 정보가 팝업으로 표시

### 3. 코호트(과정) 관리
- 과정별로 학생 데이터가 완전히 분리되어야 함
- 기수별 관리 (반도체설계 3기, 임베디드 2기 등)
- 하루에 1개 과정만 활성화

### 4. 퀴즈 & 성취도
- 수업 중 즉석으로 퀴즈를 출제하거나 문제 은행에서 불러올 수 있어야 함
- 학생별 퀴즈 성취도가 누적 관리되어야 함
- 데이터 기반으로 학생 실력을 객관적으로 파악

### 5. 팀 편성
- 성취도 데이터를 기반으로 균형 있는 팀 자동 추천
- 수동 조정 가능

## 요구사항 상세

### 피드백 버튼
- 강사가 버튼 내용을 직접 커스터마이징 가능
- 버튼 세트를 미리 저장해두고 수업 중 전환
- 버튼 스타일: 이모지 or 색상버튼 선택
- 버튼 개수: 2~5개
- 색상: 16색 팔레트
- 이모지: 5개 카테고리 (감정/손동작/기호/동물/물건)
- 기본 세트 3개 탑재

### 학생 정보
- 필수 항목: 이름, 성별, 나이, 학교, 전공, 전화번호
- 입력 방법: 직접 입력 or 엑셀/CSV 업로드 (컬럼 매핑 지원)
- 학생 직접 입력 방식: 사용 안 함

### 성취도 계산
- **퀴즈 정답률 70% + 이해도 평균 30%**
- 이해도 점수: 잘됨=100점, 애매=50점, 모름=0점, 질문=제외
- 등수: 강사 화면에만 표시 (학생에게 비공개)

### 출석
- 학생이 이름 클릭해서 접속 시 자동 출석 처리
- 미접속 학생은 결석 처리
- 학생 프로필에 출석률 표시

### 퀴즈
- 유형: OX 퀴즈, 4지선다
- 출제 방법:
  - 즉석 출제: 그 자리에서 바로 문제 생성 후 출제
  - 문제은행 출제: 과목/챕터 선택 후 랜덤 or 직접 선택
- 즉석 출제한 문제를 문제 은행에 저장 가능

### 문제 은행
- 과정별 독립 관리
- 과목 / 챕터 분류 체계
- 입력 방법: 직접 입력 or 엑셀 업로드 (전용 양식 제공)
- 이미지 첨부: 추후 추가 예정 (현재는 텍스트만)

### 교실 레이아웃
- 교실마다 책상 배치가 다름 → 드래그로 자유롭게 배치
- 수정모드: 책상 드래그 이동, 추가/삭제, 학생 배정
- 고정모드: 잠금 상태, 색상 표시만 (기본값)
- 레이아웃 교실별 저장/불러오기

### 공지
- 텍스트 공지 전송
- 링크 삽입 가능 (클릭 시 새 탭으로 열림)

### 시스템
- 배치파일로 서버 시작/종료 (브라우저 자동 오픈)
- DB 백업: 수동 버튼, 날짜별 파일 저장
- 비밀번호 변경: 설정 화면에서 가능
