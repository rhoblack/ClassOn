# ClassOn 시스템 아키텍처

## 전체 구조

```
강사 PC (Windows) — 서버 + 강사 화면
│
│  ┌─────────────┐     ┌──────────────────┐
│  │  Node.js    │────▶│   SQLite DB      │
│  │  서버 (로컬)│◀────│  classson.db     │
│  └──────┬──────┘     └──────────────────┘
│         │
│    같은 네트워크 (Wi-Fi / LAN)
│         │
├──── 강사 브라우저 (localhost:3000)
│
└──── 학생 PC들 브라우저 (강사IP:3000)
```

---

## 기술 스택

| 항목 | 기술 | 선택 이유 |
|------|------|----------|
| 서버 | Node.js + Express | Windows 설치 쉬움, 가볍고 빠름 |
| 실시간 통신 | Socket.io | WebSocket 기반 실시간 처리 |
| 데이터베이스 | SQLite | 별도 DB 서버 없이 파일 1개로 관리 |
| 프론트엔드 | HTML / CSS / JS | 브라우저만으로 동작, 설치 불필요 |
| 드래그앤드롭 | interact.js | 경량 라이브러리, 책상 레이아웃 편집 |
| 폰트 | Pretendard | 무료, 가독성 우수 |

## 모듈형 아키텍처 (확장성 설계)

### 설계 원칙
- 기능을 모듈처럼 끼웠다 뺐다 할 수 있는 구조
- core는 거의 수정 없이 유지
- 기능 추가 = 모듈 폴더 1개 추가 + config에서 true
- 기능 삭제 = config에서 false (코드 삭제 불필요)

### 폴더 구조
```
ClassOn/
│
├── core/                    ← 핵심 엔진 (거의 수정 안 함)
│    ├── server.js           서버 시작점
│    ├── socket.js           실시간 통신
│    ├── database.js         DB 연결
│    └── auth.js             인증
│
├── modules/                 ← 기능 모듈 (여기서 추가/삭제)
│    ├── feedback/           실시간 피드백
│    ├── quiz/               퀴즈 & 성취도
│    ├── students/           학생 관리
│    ├── courses/            과정 관리
│    ├── questions/          문제 은행
│    ├── teams/              팀 편성
│    ├── layout/             교실 레이아웃
│    └── system/             시스템 운영
│
├── public/                  ← 브라우저 화면
│    ├── teacher/            강사 화면 (모듈별 분리)
│    └── student/            학생 화면
│
└── config/
     └── modules.config.js   모듈 ON/OFF 설정
```

### 모듈 ON/OFF 설정
```javascript
// config/modules.config.js
module.exports = {
  feedback:  true,
  quiz:      true,
  students:  true,
  courses:   true,
  questions: true,
  teams:     true,
  layout:    true,
  system:    true,
  // 추후 추가 예정
  // attendance_report: false
  // lecture_note:      false
}
```

### 각 모듈 내부 구조 (통일된 패턴)
```
modules/quiz/
├── quiz.routes.js     API 주소 정의
├── quiz.socket.js     실시간 통신 처리
├── quiz.service.js    비즈니스 로직
├── quiz.model.js      DB 쿼리
└── quiz.config.js     모듈 설정값
```

## 데이터베이스 구조

### 설계 원칙
- 각 모듈이 자기 테이블만 소유
- 모듈 간 연결은 ID로만 참조
- 새 기능 추가 = 새 테이블 추가 (기존 테이블 수정 최소화)

### 테이블 목록
```
courses        과정 (코호트)
students       학생 기본 정보
enrollments    과정-학생 연결
feedback_logs  이해도 버튼 클릭 이력
quizzes        퀴즈
quiz_answers   학생별 퀴즈 응답
questions      문제 은행
button_sets    피드백 버튼 세트
teams          팀 편성
layouts        교실 레이아웃 (책상 좌표)
attendance     출석 기록
system_config  시스템 설정 (비밀번호 등)
```

## 실시간 통신 흐름

```
학생이 버튼 클릭
      ↓
  Socket.io 서버로 전송 (0.1초 이내)
      ↓
  서버가 강사 화면으로 즉시 전달
      ↓
  강사 책상 색상 실시간 변경

강사가 공지 전송
      ↓
  서버가 모든 학생 화면에 동시 broadcast
      ↓
  학생 화면 즉시 업데이트
```

## URL 접근 구조

```
/login              강사 로그인
/teacher            강사 메인 (로그인 필수)
  /teacher/dashboard    대시보드
  /teacher/class        수업 진행
  /teacher/students     학생 관리
  /teacher/quiz         퀴즈 출제
  /teacher/questions    문제 은행
  /teacher/teams        팀 편성
  /teacher/settings     시스템 설정
/student            학생 입장 (이름 선택)
  /student/class        피드백 화면
```

## 실행 방법 (강사 매일 루틴)

```
1. ClassOn_시작.bat 더블클릭
   → Node.js 서버 실행
   → 브라우저 자동으로 localhost:3000 열림

2. 강사 로그인

3. 오늘 수업할 과정 선택

4. 학생들: 브라우저에서 강사IP:3000 접속
          → 이름 클릭 → 입장

5. 수업 종료 후 ClassOn_종료.bat 더블클릭
```
